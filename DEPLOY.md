# ACS — Production Deployment Guide

> Step-by-step guide for deploying ACS to a VPS or shared hosting (cPanel).

---

## 1. Prerequisites

- VPS or cPanel host with **PHP 8.4+** and `php-fpm`
- **MySQL 5.7+** (or MariaDB 10.4+)
- **Node 20.x LTS** (for `npm run build` — only needed at build time)
- **Composer 2.x**
- A domain name pointed at your server
- (Optional) Redis for cache/queue
- (Optional) Supervisor for queue workers

---

## 2. Initial server setup

### 2.1 Clone the repo

```bash
cd /var/www
git clone https://github.com/wafazz/ACS-Automation-Tools.git acs
cd acs
```

### 2.2 Install dependencies

```bash
composer install --no-dev --optimize-autoloader
npm ci
npm run build
```

> **Note:** `node_modules` is gitignored — `npm ci` regenerates it.
> `npm run build` produces the production assets in `public/build/`.
> After build, `node_modules` can be deleted to save space.

### 2.3 Environment file

```bash
cp .env.example .env
php artisan key:generate
```

Edit `.env` with production values:

```dotenv
APP_NAME=ACS
APP_ENV=production
APP_KEY=base64:...                # generated by key:generate
APP_DEBUG=false                   # NEVER true in production
APP_URL=https://your-domain.com

LOG_CHANNEL=daily
LOG_LEVEL=warning

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306                      # 3306 in prod, NOT 3307 (XAMPP local only)
DB_DATABASE=acs_production
DB_USERNAME=acs_user
DB_PASSWORD=                      # strong password, no quotes needed unless special chars

# Cache & session — use database for simplest setup, redis for performance
CACHE_STORE=database
SESSION_DRIVER=database
QUEUE_CONNECTION=database         # or 'redis' if Redis is available

# Mail — leave default; Brevo overrides via DB settings (see Phase 10.5)
MAIL_MAILER=log

# Billplz fallback (admin can override via DB at /admin/settings/billplz)
BILLPLZ_SANDBOX=false             # production!
BILLPLZ_API_KEY=
BILLPLZ_X_SIGNATURE=
BILLPLZ_COLLECTION_ID=

VITE_APP_NAME="${APP_NAME}"
```

### 2.4 Create the database

In MySQL:

```sql
CREATE DATABASE acs_production CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'acs_user'@'localhost' IDENTIFIED BY 'strong-password-here';
GRANT ALL PRIVILEGES ON acs_production.* TO 'acs_user'@'localhost';
FLUSH PRIVILEGES;
```

### 2.5 Run migrations + seeders

```bash
php artisan migrate --force
php artisan db:seed --class=TemplatePackSeeder --force   # only the catalog packs, not the demo user
```

> `--force` is required in `production` env.
>
> **Skip the full DatabaseSeeder in production** — it creates a `demo@acs.local` test account with 25 fake leads. Use it only on staging/local.

### 2.6 Create the first admin

```bash
php artisan tinker
```

In tinker:

```php
\App\Models\User::create([
    'name' => 'Admin',
    'email' => 'you@yourdomain.com',
    'password' => bcrypt('temporary-password-change-me'),
    'phone' => '0123456789',
    'industry' => 'other',
    'plan' => 'founder_ltd',
    'is_admin' => true,
    'email_verified_at' => now(),
]);
```

Change the password on first login from `/profile`.

### 2.7 Storage symlink + permissions

```bash
php artisan storage:link
chown -R www-data:www-data storage bootstrap/cache
chmod -R 755 storage bootstrap/cache
```

(On cPanel, use the panel's user — usually your cPanel username — instead of `www-data`.)

## 3. Web server config

### 3.1 Nginx (VPS)

`/etc/nginx/sites-available/acs`:

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/acs/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;
    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.4-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

Enable + reload:

```bash
sudo ln -s /etc/nginx/sites-available/acs /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3.2 cPanel (shared hosting)

1. Upload the project to `~/acs/` (or wherever).
2. Point your domain's document root to `~/acs/public/`.
3. In **MultiPHP Manager**, set the domain to PHP 8.4.
4. Use cPanel's **Terminal** or **SSH** for `composer install` + `npm ci && npm run build`.

## 4. SSL (Let's Encrypt)

### VPS

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

Cert auto-renews via certbot's systemd timer.

### cPanel

Use **AutoSSL** under SSL/TLS Status — toggle it on for the domain.

## 5. Cron (Laravel scheduler)

The scheduler runs trial-expiry warnings (and any future scheduled tasks).

Add to crontab (`crontab -e`):

```
* * * * * cd /var/www/acs && php artisan schedule:run >> /dev/null 2>&1
```

On cPanel: **Cron Jobs** → add a new entry with command `cd ~/acs && php artisan schedule:run >> /dev/null 2>&1` running every minute.

Verify scheduled tasks:

```bash
php artisan schedule:list
```

## 6. Queue worker (optional but recommended)

ACS uses the database queue driver by default — no extra service needed if you don't queue anything heavy. To process queued jobs (e.g. future webhook side-effects):

### VPS — supervisor

`/etc/supervisor/conf.d/acs-queue.conf`:

```ini
[program:acs-queue]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/acs/artisan queue:work --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
user=www-data
numprocs=1
redirect_stderr=true
stdout_logfile=/var/log/acs/queue.log
stopwaitsecs=3600
```

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start acs-queue:*
```

### cPanel

Use a cron job that runs every 5 minutes:

```
*/5 * * * * cd ~/acs && php artisan queue:work --stop-when-empty >> /dev/null 2>&1
```

## 7. Wire integrations (one-time, in admin panel)

After first deploy, log in as the admin user you created in step 2.6 and visit:

1. `/admin/settings/billplz` → paste your **production** Billplz keys → Save → click "Test Connection"
2. `/admin/settings/brevo` → paste your Brevo API key + verified sender email → Save → click "Send Test Email"
3. `/admin/settings/onsend` → paste your Onsend instance ID + access token → Save → click "Send Test WhatsApp"

You can also set them in `.env` as fallbacks (BILLPLZ_API_KEY, etc.).

### Billplz webhook URL

In your Billplz Collection settings, set the callback URL to:

```
https://your-domain.com/billplz/callback
```

This route is CSRF-exempt and validates the `x_signature` HMAC.

## 8. Backup strategy

### Daily MySQL dump

Add to crontab:

```
0 3 * * * /usr/bin/mysqldump -u acs_user -p'PASSWORD' acs_production | gzip > /var/backups/acs/db-$(date +\%Y\%m\%d).sql.gz
0 4 * * * find /var/backups/acs/ -mtime +30 -type f -delete
```

For offsite backup, pipe to S3/B2/rsync:

```
0 3 * * * mysqldump ... | gzip | aws s3 cp - s3://your-bucket/acs/db-$(date +\%Y\%m\%d).sql.gz
```

### Storage backup

Just `storage/app/` (uploads) + `.env`:

```bash
0 3 * * * tar -czf /var/backups/acs/storage-$(date +\%Y\%m\%d).tar.gz -C /var/www/acs storage/app .env
```

## 10. Post-deploy sanity checks

```bash
# Routes registered
php artisan route:list

# Migrations applied
php artisan migrate:status

# Scheduler can run
php artisan schedule:list

# Cache + config compiled (production speedup)
php artisan config:cache
php artisan route:cache
php artisan view:cache

# (Re-run after any .env change)
```

In the browser:

1. `https://your-domain.com` — landing renders
2. Register a test user → check email arrives (if Brevo configured)
3. `/login` as admin → sidebar shows ADMIN section with badges
4. `/admin` → MRR / users / leads counts load
5. Add a lead → 3 reminders auto-spawn (Day 1/3/7)
6. Open DevTools → Application → Service Workers → confirm `sw.js` is installed
7. iPhone Safari → Add to Home Screen → confirm app opens standalone

## 11. Common gotchas

| Symptom | Likely cause | Fix |
|---|---|---|
| 500 on every page after deploy | `APP_KEY` missing | `php artisan key:generate` |
| Vite assets 404 | Forgot `npm run build`, or `public/build/` not deployed | Re-build, ensure dir is uploaded |
| 403 on Inertia POSTs | Missing CSRF cookie / session driver mismatch | Verify `SESSION_DRIVER` + cookies enabled |
| Billplz webhook returns 419 | Route forgot CSRF exemption | Already exempted in `bootstrap/app.php`, redeploy |
| Trial warnings never fire | Crontab missing | Add `* * * * * cd /var/www/acs && php artisan schedule:run` |
| `composer install` fails on PHP version | `composer.lock` was generated on different PHP version | `rm composer.lock && composer install` |
| Mailbox not receiving Brevo emails | Sender email not verified in Brevo account | Verify the domain/email in Brevo dashboard |
